Syntax

Use two space indentation. This is the best way to guarantee that code is rendered the same no matter the environment.

  • Editors should be programmed to perform a 'soft tab' equal to two spaces if the developer prefers to use the tab key.
  • Major file-size issues related to using spaces instead of tabs are inconsequential because of minification standards.

When grouping together, Selectors should be placed on their own lines and separated by a comma. Note: related selectors should be placed on the same line.

One space should be used between selectors and the opening brace of the declaration block. In addition, closing braces should be placed on a new line by themselves.

Use double quotes when using attribute values in selectors.

Declarations should always be indented once (2 spaces).

A space should be placed after the : for each declaration and each declaration should be on its own line (unless it is a single declaration - if this is the case, see - Single Declarations).

Each declaration should end in a semi-colon (;).

Prefix property values with a leading zero based on the following ruleset:

  • If a value has the potential to exceed 1.0, use the leading zero ( rem, em, %)
  • If a value can never exceed 1.0, do not use the leading zero. (opacity)

When assigning a zero value to a property, avoid specifying the unit of measurement (margin: 0; instead of margin: 0px;)

Include a space after each comma in separated property values (rgba(), box-shadow(), etc.)

When writing declarations that are related to each other, attempt to align their values for easier editing.

// Proper syntax .selector, .selector-two, .selector-two--modifier, .selector-three { // styles } .selector[type="..."] { // styles } .selector { color: #fff; font-size: 1rem; margin-top: 0.9rem; margin-bottom: 0; background-color: rgba(0, 0, 0, .5); } .selector-two { -webkit-border-radius: 3px; -moz-border-radius: 3px; border-radius: 3px; } .selector-three { position: absolute; top: 0; right: 0; bottom: 0; left: 0; }
