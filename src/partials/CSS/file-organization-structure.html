File Organization/Structure

Preferred SASS & LESS file organization is using ITCSS (Inverted Triangle CSS) methodology. A boilerplate and introduction of this folder structure can be found here.

Code should be split into multiple files in order to keep a clear separation of components.

Do not organize code by page as this narrows a component's scope drastically.

Ensure that every file has an introductory comment with at least the stylesheets name or what it contains.

Use consistent white space between rule sets in order to increase legibility. Preferred white space is one blank line.

// White space .selector { // styles } .selector-two { // styles } .selector-three { // styles }
