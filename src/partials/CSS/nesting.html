Nesting

Do not nest styles unless you are scoping those styles specifically to a parent. Although nesting may appear to keep code organized in SASS or LESS it causes the compiled CSS code to become unreadable. In addition, nesting can cause specificity issues throughout the project. Rule: Don't nest unless you absolutely have to

Consider creating styles in a more modular fashion in order to reduce or eliminate the need for nesting.

If nesting is necessary, avoid going more than 2 levels deep.

When nesting styles, leave a blank line before and after the nested ruleset.

// Avoid .selector div p span { // styles } .selector-two { div { p { span { // styles } } } } // Instead .selector { // styles } .div { // styles p { // styles } } span.selector { // styles }
