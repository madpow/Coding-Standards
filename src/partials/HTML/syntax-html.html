Syntax

Use two space indentation. This is the best way to guarantee that code is rendered the same no matter the environment.

  • Editors should be programmed to perform a 'soft tab' equal to two spaces if the developer prefers to use the tab key.
  • Major file-size issues related to using spaces instead of tabs are inconsequential because of minification standards.

Nested elements are always indented once.

Use double quotes when assigning attribute values to elements.

Use minimal markup to present content, styles, and any interaction required.

Avoid using <br> tags to separate paragraphs or content. Use <p> with properly configured margins or style using CSS in order to overcome this extra markup.

Do not include trailing slashes for self-closing elements. Do include closing tags for those elements where it is considered optional (<li>, <body>)

Use native, semantic HTML5 tags whenever possible for cleaner markup and generally good accessibility.

Do not use tables for layout. Only use tables when displaying tabular data.

Structure blocks of code in a modular fashion so that they can be reused throughout projects during implementation. Refer to Brad Frost's Atomic Design as a reference for this style of coding.
